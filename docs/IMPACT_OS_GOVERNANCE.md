## 9. User Roles & Access Model

### The 4 Human User Categories

| Role | Type | Core Function |
|------|------|---------------|
| **Participant (Player)** | Human | Acts, fails, earns |
| **Mentor / Instructor** | Human | Teaches, guides |
| **Admin / Operator** | Human | Operates system |
| **Impact Officer (M&E)** | Human | Measures outcomes |

### The System Actor

| Role | Type | Core Function |
|------|------|---------------|
| **System (AI + Rules Engine)** | Non-human | Enforces behavior |

### Role Details

#### Participant (Player)

- No prior skills or income required
- Progresses through identity levels
- Completes missions
- Earns currencies
- Submits income proof

#### Mentor / Instructor

- Reviews escalations only
- Unlocks paths
- Intervenes when system flags risk
- **Does NOT:** Track attendance, police behavior, decide graduation, control stipends

#### Admin / Program Operator

- Manages cohorts
- Oversees stipends (within system rules)
- Reviews income verification
- Monitors system health
- **Constraint:** Cannot override core laws without audit trails

#### Impact Officer (M&E)

- Monitors impact metrics
- Analyzes conversion rates
- Validates reports for funders
- Audits outcomes and integrity
- **Does NOT:** Teach, mentor, intervene in behavior

## 13. Failure Classification System

### System Position on Failure

> **Failure â‰  Inaction. Inaction is the only unacceptable state.**

This is **enforced in code**, not culture.

### Failure Taxonomy

| Failure Type | Meaning | System Response |
|--------------|---------|-----------------|
| **Market Rejection** | Explicit "no" from prospect | +Arena Points, log insight |
| **Non-Response** | No reply to outreach | Adjust channel strategy |
| **Offer Declined** | Pricing/value mismatch | Pricing mission triggered |
| **Missed Deadline** | Capacity issue | Time management intervention |
| **Avoidance** | No attempt made | Warning + escalation |
| **Burnout Dip** | Engagement drop pattern | Survival check triggered |

### Failure Logging Triggers

When failure is logged:

1. Award Arena Points (for exposure-based failures)
2. Generate micro-reflection prompt
3. Adjust mission difficulty if needed
4. Update predictive models

> **Failure is data, not shame.**
